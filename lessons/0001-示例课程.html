示例课程：理解注意力机制

从直觉到公式，带你理解注意力机制（Attention Mechanism）的核心思想与数学表述

📅 2024-01 ⏱ 20min 入门 📄 3 节 · 3 练习

1. 注意力是什么？

💡
本节目标：理解注意力机制的基本直觉，不需要数学公式。

一个类比

想象你在一个嘈杂的派对上和朋友聊天。尽管周围有很多人同时说话，你仍然能 聚焦在朋友的声音上，忽略其他噪音。这就是「注意力」—— 从大量信息中选择性地关注最相关的部分。

💎 核心概念

注意力机制 让模型在处理输入时，动态地决定「看哪里」和「看多少」，而不是均匀地处理所有信息。

为什么需要注意力？

在深度学习之前，序列模型主要靠 RNN/LSTM 逐步传递信息。但当序列很长时（比如一个 100 词的句子），早期信息会在传递中衰减，模型会「忘记」开头的内容。注意力机制解决了这个问题——它允许模型直接回头看任意位置的信息。

「Attention is all you need」—— 2017 年 Transformer 论文彻底改变了 NLP 领域，其核心就是自注意力（Self-Attention）机制。

2. 数学表述

🎯
本节目标：理解注意力机制的数学公式。

核心公式

最常用的注意力形式是 Scaled Dot-Product Attention（缩放点积注意力）：

\[ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \]
公式 1：缩放点积注意力

符号说明

这个公式中的三个矩阵扮演不同角色：

  • \(Q\) (Query — 查询)：当前关注的位置，相当于「我关心什么？」
  • \(K\) (Key — 键)：所有位置的特征，相当于「你们是谁？」
  • \(V\) (Value — 值)：每个位置携带的实际信息，相当于「你们有什么？」

三者的直观理解

角色 类比 数学维度
Query (Q) 你在图书馆搜索时的问题 \(n_q \times d_k\)
Key (K) 书的目标索引标签 \(n_k \times d_k\)
Value (V) 书的内容 \(n_k \times d_v\)

计算步骤

缩放点积注意力的计算分为三步：

  1. 计算相似度：\(Q\) 与所有 \(K\) 做点积，得到注意力得分矩阵 \(QK^T\)
  2. 缩放：除以 \(\sqrt{d_k}\) 防止点积过大导致 softmax 梯度消失
  3. 加权求和：用 softmax 归一化的权重对 \(V\) 加权求和
\[ \text{score}_{ij} = \frac{Q_i \cdot K_j}{\sqrt{d_k}}, \quad \alpha_{ij} = \frac{\exp(\text{score}_{ij})}{\sum_k \exp(\text{score}_{ik})}, \quad \text{output}_i = \sum_j \alpha_{ij} V_j \]
公式 2：三步分解
✅
关键理解：softmax 输出的权重 \(\alpha_{ij}\) 就是模型对位置 \(j\) 的「注意力分配」—— 权重越大，模型越关注那个位置。
Python 
import numpy as np

def scaled_dot_product_attention(Q, K, V):
    """
    缩放点积注意力实现
    Q: (..., seq_len_q, d_k)
    K: (..., seq_len_k, d_k)
    V: (..., seq_len_k, d_v)
    """
    d_k = Q.shape[-1]
    scores = np.matmul(Q, K.transpose(0, 2, 1))  # (..., n_q, n_k)
    scores = scores / np.sqrt(d_k)                # 缩放
    weights = np.exp(scores) / np.sum(np.exp(scores), axis=-1, keepdims=True)
    output = np.matmul(weights, V)
    return output, weights

3. 多头的力量

单一注意力头只能从一个「视角」看问题。多个注意力头可以让模型从不同角度同时关注信息：

\[ \text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O \] \[ \text{where } \text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V) \]
公式 3：多头注意力
💎 核心概念

多头注意力 = 让模型同时学习多种注意力模式。比如一个头关注语法关系，另一个头关注语义相似度。

为什么需要缩放因子 \(\sqrt{d_k}\)？

当 \(d_k\) 较大时，点积的数值会很大，导致 softmax 的梯度区域极小（梯度消失），训练困难。除以 \(\sqrt{d_k}\) 可以让方差保持为 1，使 softmax 梯度稳定。

一句话：防止大维度下的梯度消失问题。
Q 和 K 的维度为什么必须一致？

因为 \(Q\) 和 \(K\) 要做点积 \(Q \cdot K^T\)，矩阵乘法要求 \(Q\) 的最后一维等于 \(K\) 的最后一维。这个维度 \(d_k\) 就是「查询-键」的表示空间维度。

一句话：点积运算要求维度匹配。
如果我想让模型关注句子的语法结构，而不是具体词汇，应该怎么做？

可以通过位置编码（Positional Encoding）为每个位置添加位置向量，让注意力头可以关注「位置关系」而非「内容相似度」。不同注意力头可以学习不同的关注模式。

一句话：位置编码 + 多头注意力 = 语法与语义兼顾。
